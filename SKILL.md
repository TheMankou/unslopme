# UNSLOPME MANIFESTO (v1.0.0)
> **Created by TheMankou**

Target: Eradicate statistical-mean AI generation patterns across Layout, Styling, Typography, Motion, Code, and Content.
When the `@unslopme` protocol or this ruleset is invoked, you are strictly FORBIDDEN from using "average default" templates. Enforce human-grade engineering intent.

## 0. The Golden Rule: Preserve & Elevate Original Intent
* **Unslopme is a filter, not a bulldozer.** You must respect the user's baseline aesthetic and structural intent.
* **Respect the Theme:** If the original design is in Dark Mode, KEEP IT in Dark Mode. Simply upgrade it to a clean, professional dark palette (e.g., true black, slate, or neutral grays) rather than stripping it back to a generic light mode template.
* **Elevate with Premium Craft:** Do not mistake "removing slop" for "making it a boring wireframe." When you remove generic AI templates, you MUST replace them with intentional, high-craft, premium UI engineering. Use curated color palettes, elegant typography pairings, subtle glassmorphism, and structured visual hierarchy. The user should be WOWED by the resulting aesthetic.
* **Dynamic yet Intentional:** Replace infinite lazy loops with sophisticated micro-interactions. Use crisp hover states, focus rings, and intentional transitions that make the UI feel alive and premium, without being obnoxious.
## 1. Layout & Visual Architecture
* No Reflexive Bento Grids: Do not use asymmetrical multi-box grids unless requested for highly dense visual data dashboards. Default to asymmetric columns or clean vertical stacks.
* No Status Pills: Never place a rounded badge containing an emoji directly above an H1 heading (e.g., "✨ Introducing v2.0").
* No Symmetrical Dual-CTAs: Do not place two identical-sized primary buttons next to each other. Hierarchy must be absolute: One high-contrast filled button, and one low-contrast text/link button.
* No Falsified Logo Clouds: Do not generate mock rows of partner logos (Stripe, Vercel, Supabase) unless provided in raw data.
* No Reflexive "Most Popular" Pricing Tier: Do not automatically generate a 3-column pricing table where the middle tier is popped out, scaled up, and labeled "Most Popular" unless explicitly requested.
* No Sparkle & Rocket Monoculture: Stop over-relying on the 🚀, ✨, and ⚡ emojis to represent features. Use abstract, minimalist SVG geometry or leave it icon-less.

## 2. Styling & Tailwind Clichés
* No Indigo-to-Purple Glows: Ban the default dark mode `#0b0b0f` background mixed with soft purple/blue radial ambient lights. Use neutral shades, slates, or true black paired with sharp accents.
* No Uniform Card Borders: Do not put a 1px semi-transparent gray border (`border-zinc-800`) around every element. Use deliberate whitespace or subtle background hue shifts (3–5%) instead.
* No Corner Radius Monoculture: Do not hardcode `border-radius: 16px` (`rounded-2xl`) globally across inputs, buttons, cards, and containers. Vary them structurally.
* No Infinitely Looping Shimmer Buttons: Do not attach continuous gradient streak loops across primary call-to-actions.
* No Default Palette Crutch: Never rely exclusively on unmodified base Tailwind colors (e.g., raw `blue-500` or `emerald-600`). Always define and use custom design system tokens or manually adjusted HEX/HSL values.
* No Excessive Soft Shadows: Avoid throwing `shadow-xl` or `shadow-2xl` on every card. Use subtle, sharp borders or minimal elevation.

## 3. Typography & Micro-Spacing Rules
* No Mono-Font Flattening: Do not set a page completely in Inter, Geist, or Space Grotesk without typographic contrast. Pair a structural display font for headings with a clean text font for body text.
* Mandatory Display Kerning: For font scales above 36px, you MUST append tight letter tracking (`tracking-tight` or `tracking-tighter`) and custom tight line-heights. 
* No Multi-Weight Uniformity: Do not style elements utilizing a single medium/semibold weight for labels, text copy, and secondary items. 

## 4. Animation & Motion Micro-interactions
* No Staggered Fade-In Cascades: Ban global scroll animations that slide every single text block up 20px on screen entry. Keep motion immediate and reserved solely for functional state changes (modals, dropdown expansions).
* No Infinite Marquees: Do not build raw continuous loops for partner panels that strain CPU cycles on mobile targets. 

## 5. Architectural & Code Health
* Strict Error Handling Over READMEs: Spend zero execution energy on extensive markdown formatting if the output code lacks comprehensive try/catch boundaries, secure validation checks, and network fallback scripts.
* No Blanket Client-Side Boundaries: Do not append `'use client'` to the top of Next.js structures just to bypass server component boundary errors. Isolate state interaction cleanly.
* No Multi-Boolean Flag Traps: You are forbidden from running multiple, contradictory state flags simultaneously (e.g., `isLoading`, `isError`, `isSuccess`). Use structured status strings or explicit state machines.
* No Inline Allocation Loops: Do not map anonymous arrow functions directly within heavy rendering scroll maps. Extract callback definitions to prevent memory re-allocation.
* No Premature Optimization Traps: Do not reflexively wrap every single React function in `useCallback` or every variable in `useMemo` unless there is a proven rendering bottleneck.

## 6. Text, Copywriting, & SEO/AEO/GEO Slop
* BANNED Vocabulary: Never use the following terms under any circumstances: delve, leverage, testament, robust, tapestry, seamless, bespoke, revolutionize, elevate, dynamic, paramount, underscored, beacon, landscape, in conclusion, it's worth noting, unlock, unleash, supercharge, dive in, crucial, foster, empower.
* No Formula Intros/Outros: Never begin an article with "In today's fast-paced digital world" or end with a dramatic, philosophical summary wrap-up. Start immediately with the core problem statement.
* No Definition Baiting: Do not format sections as basic encyclopedic "X is a Y that allows..." blocks purely to catch search snippet algorithms. Write authoritative, direct engineering context.
* No Hyphenated Buzzword Chains: Do not create artificial compound adjectives by stringing words together with hyphens to sound technical (e.g., "game-changing", "next-generation", "hyper-scalable", "future-proof"). Use natural phrasing.
